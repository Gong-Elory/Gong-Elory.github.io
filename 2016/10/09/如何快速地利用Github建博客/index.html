如何快速地利用Github建博客

发表于 2016-10-09   |   分类于 Github实用   |     |   阅读次数


作为一个小白，想拥有自己的独立域名博客，想想都是一件刺激的事情，但是想想那些令人头疼且繁杂的步骤是不是让很多人都望而却步了呢?
肯定有很多人会百度谷歌：如何快速用Github建立博客？ 用什么模板系统？简不简单？网上的答案基本上将这些问题根据不同层次的人描述得的很清楚了。笔者今天就将自己建站过程中，看到的一些好的建站步骤博文给大家罗列出来，让大家少走弯路，节省时间，把更多的精力用在创作优秀的文章上。

笔者将按这个流程讲解：

  • 购买独立域名，域名解析相关工作
  • 注册Github账户，建立GitHub Pages博客
  • 域名绑定
  • 博客配置
  • 文章撰写


看到这里，千万别慌，一步步做，很简单的。



Step1. 域名购买，域名解析

所谓域名，其实解释给IP地址带上一个便于记忆的标志。譬如我们去某宝购物，输入的taobao.com就是一个域名，没人会去记200.210.6.20（随便编的）这一串IP。

去哪里买呢？很多地方有卖，学生可以去阿里云看看，学生购买域名和租服务器有优惠。笔者在Godday购买，网络上有很多Godday的优惠码，点击网页里面的优惠码链接，然后购物，结算的时候会自动优惠，大部分支持支付宝支付。域名后缀有很多种，常见的 .com .net``.org。喜欢特别的可以像笔者一样.me等等，自己喜欢就好。

然后选择DNSPod的解析服务，口碑挺好的。在解析这块,Beiyuu的博文头部有具体流程。值得注意的是：Github Pages的服务IP地址会变化，具体的请点击链接查看。目前的是：192.30.252.153和192.30.252.154。

Step2. 注册Github账户，配置使用Git，建立GitHub Pages博客

来看这篇文章的，应该都会注册用户，github也是一样，很普通的注册方法。点此注册.

注册后，安装Git，点此安装Windows版，其他版本请点击Mac，Linux.如果你会一点点基本的嵌入式命令行，学起来毫不费劲，看一下伯乐在线的这篇博文，跟着敲一敲，很快就get了。如果没有基础的话,可以跟着伯乐在线的速学一下，想深入的话，点这里Git中文教程学习。掌握了一些基本命令行之后，按照Beiyuu博客的配置和使用Github部分进行相关设置。

建立Github博客,可以看Ryane博客的将博客部署到Github Pages上部分。值得注意的是：建立的Repository名：username.github.io中的username名必须与左侧你的用户名Owner保持一致。

Step3. 域名绑定

做完上面一步，你可以通过username.github.io这个域名访问到你的测试博客。此时要想通过你自己的购买的域名访问，就必须绑定。

首先：在Github->username.github.io这个repository中新建一个文件（Create new file），文件名叫CNAME，没有后缀，在首行写上你买的域名，譬如笔者的域名gelory.me。

然后：登录你注册了的DNSPod，添加一条如图所示的记录：

值得注意的是：记录值：username.github.io.io后面这个点千万别忘了。添加完后，等上个几分钟，就绑定好了。当你通过username.github.io这个域名访问测试博客时，他会自动跳转到你自己的域名上。

Step4. 博客配置

到这一步，你已经可以访问域名了，只不过博客还没成型，你需要配置。对于小白来说，采用Hexo框架再好不过了。

可以看Ryane博客的Hexo部分。
值得注意的是:当你把本地文件push到github上时，可能会将你刚才新建的CNAME文件删掉，那么你就无法通过域名访问了。`解决方法是: 在 \hexo\source文件夹下，重新建立CNAME文件。

Step5. 文章撰写

如何编辑文章，请继续查看Ryane博客的在博客写文章部分。格式化编辑文章，大家都在推荐使用MarkDown，这东西确实不错，我用的编辑器是作业部落的CMD MarkDown非常好使，在此推荐。另外，会一点前端知识能让你的文章样式更加丰富。

###小样有话说：

到此，你的博客建完了，文章也会写了。那请你别辜负了当初建站的热情，把心思放在创作优质内容上。加油！
