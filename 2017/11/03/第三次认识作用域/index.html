第三次认识作用域

发表于 2017-11-03   |   分类于 前端   |     |   阅读次数

作用域的理解深化

什么是作用域：

作用域规定了变量或者函数可访问的范围，js采用的是词法作用域（静态作用域）规则，也就是说作用域在函数被定义的时候就已经生成。
这个概念喜欢和上下文搞混。我认为上下文规定了一个环境，一个函数调用或者变量访问的环境，上下文其实可以实体化，这样便于加深印象，我们可以把上下文看作一个对象，这个对象有很多属性，其中有一个this，this就指向这个对象本身。我们知道this只有在调用时才知道自己指向谁，所以说一个上下文环境也只有在js执行的时候才能明确。譬如一个函数被调用的时候我们才能确定他的上下文。我们也可以通过bind，apply，call改变这个函数的上下文。

什么是作用域链：

当查找某个变量的时候，会首先在当前上下文的变量对象中找，如果没有，则会从父级上下文的变量对象中查找，一直找到全局上下文的变量对象，如果还没有，就报错。这种有多个上下文的变量对象构成的变量对象链表就叫做作用域链。所以所谓的作用域链其实就是一个指向多个变量对象的指针列表，他只引用但是不包含变量对象。函数被调用之后，退出该执行环境，如果其变量对象没有被引用的话就被垃圾回收机制回收。闭包就是利用了这点。闭包说白了就是一个可以从外部访问函数内部的机制，其原理就是因为一直引用着父作用域的变量对象，导致其没有被回收，所以会一直存在，可供使用 。所以最好的方法就是使用完后，手动清除。

作用域链怎么生成的：

函数在被定义的时候，有个内部属性[[scope]]，这个属性保存的是一条作用域链，其实就是父作用的[[scope]]+父作用域的AO，但是不完整。当函数被调用时，在js创建阶段会生成一个变量对象，并且把这个变量对象保存到刚才说的那条作用域链上，最前端。同时还创建了一条新的表示函数调用作用域的链 scope = [[scope]] + VO
所以此时就有了两条作用域链，看似好像是相同的，其实就是相同但不相等的两条作用域链。
个人认为（个人理解不代表正确观点）：
第一条作用域链的作用是用于函数内部函数创建时用于填充[[scope]]的，就跟这个外部函数的[[scope]]是由其’父函数’作用域链填充一样（如果’父函数’为全局执行环境，那么外部函数的[[scope]]就只包含一个AO全局执行环境的变量对象。）。
第二条作用于链用于函数调用。
